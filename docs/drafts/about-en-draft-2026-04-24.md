## Section A: What Is Synapse

**Eyebrow**: `The Product`
**Heading**: `A synapse connects signals. So does this.`

Synapse is an AI team operating system — the connective layer between knowledge (what the team knows), decisions (how it picks), and execution (what it does). Named for the neural junction that passes signals between neurons, because that's the function it serves: the place where information flows reliably from one stage of work to the next.

Built on Harness Engineering — the 2026 paradigm that says a reliable agent is a model plus its harness, and the harness is where the engineering actually lives.

Synapse runs in production. Not in a demo, not in a deck. Ten teams, fifty agents, continuously audited, operating inside an enterprise delivering digital services across the construction asset lifecycle.

_(~130 words)_

## Section B: System Architecture

**Eyebrow**: `How It's Built`
**Heading**: `Five layers, one loop.`

```
Synapse — Five-Layer Architecture

├── Memory Layer     ── Obsidian Second Brain (OBS)
├── Control Layer    ── Harness Engineering (Guides + Sensors)
├── Execution Layer  ── 10 teams, 50 AI agents
├── Evolution Layer  ── Intelligence loop (daily → review → act → report)
└── Decision Layer   ── L1 auto → L2 expert → L3 CEO → L4 president
```

Each layer has one job. The memory layer remembers. The control layer binds behavior. The execution layer ships. The evolution layer keeps the system learning. The decision layer makes sure the right call lands at the right level — which is the difference between a team and a mob.

_(~120 words)_

## Section C: Core Philosophy

**Eyebrow**: `What We Believe`
**Heading**: `The model isn't the moat.`

AI competitive advantage isn't in the model itself. It's in the environment you build around it — the constraints, the feedback loops, the roles, the audit trail. The same model, harnessed differently, produces radically different teams.

> **Agent = Model + Harness.**
> **Models are a commodity. The harness is where the moat lives.**

This isn't rhetoric. It's the operating assumption every Synapse design decision rolls up to.

## Section D: What Synapse Is Not

**Eyebrow**: `Clarifications`
**Heading**: `What Synapse isn't.`

- **Not a model provider.** Synapse wraps any capable model. We don't train foundation models, we don't want to.
- **Not a no-code agent platform.** If a visual flow-builder is what you need, Synapse is the wrong shape. It's a methodology plus framework, installed by operators who write code.
- **Not a general consultancy.** We have one methodology (Harness Engineering), one delivered product (Synapse), and one industry anchor (AEC). Everything outside that is referrals.
- **Not proprietary software you rent.** The installed Synapse instance is yours. Documentation, agents, configuration — you own the repo after Implementation.

## Translation Choices & Rationale

1. **"Synapse 是什么" → "An operating system for AI teams."**
   Literal translation ("What is Synapse") is fine but generic. English hero leads with a category claim — "operating system" is a deliberate frame that positions Synapse alongside Linux and Kubernetes rather than alongside ChatGPT wrappers.

2. **"突触（Synapse）是神经元之间传递信号的关键节点" → "A synapse connects signals. So does this."**
   The Chinese explanation of the name is useful pedagogy. English rewrites tighter, one declarative plus one rhetorical echo. Readers who don't know what a synapse is will Google it; readers who do will appreciate the brevity.

3. **"刘子杨" → "Lysander Liu (刘子杨)".**
   English-first audiences need a pronounceable handle. Dual-name format keeps the Chinese name visible (important for bilingual trust signal) while giving English speakers something to say.

4. **Janus Digital: named, but not centered.** Chinese page mentions construction digital delivery as origin without naming the firm. English draft names Janus Digital explicitly as the anchor case — once, at the end of the Origin section. This follows the task brief: "Janus Digital mentioned but not core identity (Synapse as protagonist)."

5. **"What Synapse Is Not" section — new, English-only.** Chinese About pages don't typically do anti-pattern sections; English B2B software About pages do, because skeptical English-first readers trust products that say what they're not. Added four bullets to preempt the common misreads.

6. **"一天内通过与 AI CEO 的对话，从零构建了..." → "Synapse was designed...during a single working day — in continuous conversation with an AI CEO."**
   The Chinese original treats the one-day origin as casual biography. English rewrite centers it as the core origin story. It *is* the most differentiated thing about Synapse — a one-day origin is a provable, memorable, repeatable claim.

## Open Questions

1. **Janus Digital naming: correct decision?** Draft names Janus once, at the end, positioned as "anchor case." Alternative: keep it unnamed ("an enterprise operator in construction digital services"). Brand call. Same question arose on homepage — worth deciding once and propagating.
2. **"Built in a day" — marketing liability?** The one-day origin is true, memorable, and differentiating. It's also skeptic-bait ("you threw together a fifty-agent system in a day?"). Current draft frames it as design+iteration, which is accurate. Do we want an explicit footnote clarifying "design completed in one day; calibration ongoing"? Style_calibrator preference.
3. **"What Synapse Is Not" — tone check.** Four negations in a row is assertive. Some English readers love it, some find it defensive. Test with style_calibrator; could be reduced to three if needed.
4. **Architecture diagram: update format?** Currently ASCII tree. Works in code-savvy English contexts. Some enterprise About pages prefer a visual SVG. Design decision.
5. **Founder section contact.** Only one email listed. Do we want (a) a form, (b) a LinkedIn link, (c) separate contacts for sales vs. partnership vs. press? Probably (a) + (b) at minimum for a public site — RD team.
