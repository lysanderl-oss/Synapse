# 内容管线

博客内容从工作记录到发布的完整流程。

## 目录结构

- `_inbox/` — 原始想法/问题片段（未处理）
- `_drafts/` — 进行中的博客草稿
- `_published/` — 已发布文章存档
- `_templates/` — 笔记模板

## 流程

工作日志（00-daily-work/）→ 洞察笔记（_inbox/）→ 博客草稿（_drafts/）→ 发布（lysander.bond）→ 存档（_published/）

## 内容类型

| 类型 | 说明 | 频率 |
|------|------|------|
| A 系统拆解 | 具体机制拆解 + 设计动机 | 每两周 |
| B 问题日志 | 真实问题 → 分析 → 解法 | 每周2-3篇 |
| C 方法论提炼 | 可复用思维框架 | 每月1-2篇 |
| D 进化记录 | 系统迭代历史 | 每月1篇 |
