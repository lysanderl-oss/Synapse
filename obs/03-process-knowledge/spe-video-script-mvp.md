# SPE 快速入门 — MVP 视频分镜脚本

## 视频元信息

| 项目 | 内容 |
|------|------|
| **标题** | SPE 快速入门：4 个命令掌控你的每一天 |
| **时长** | 4 分 00 秒（目标） |
| **形式** | 屏幕录制 + AI 旁白（无数字人） |
| **风格参考** | Fireship / NetworkChuck — 干净终端 + 专业旁白 + 侧边信息图 |
| **目标受众** | 已使用 Synapse 体系的用户，需要了解 SPE 个人任务管理功能 |
| **语言** | 中文旁白，终端命令为英文 |

## 学习目标

看完本视频后，观众能够：

1. 理解 SPE 在 Synapse 体系中的定位（团队管理之外的"个人管理"模块）
2. 使用 `/capture` 随时捕获任务和想法到收件箱
3. 使用 `/plan-day` 生成每日焦点计划并完成收件箱分流
4. 使用 `/time-block` 为重要任务在 Google Calendar 上创建时间块
5. 使用 `/weekly-review` 生成每周回顾报告，形成完整闭环

## 教学设计说明（ADDIE 模型）

### 内容筛选

SPE 使用指引共 7 章，本 MVP 视频聚焦以下内容：

| 章节 | 是否覆盖 | 理由 |
|------|----------|------|
| 一、体系概览 | 精简覆盖（30秒） | 建立上下文，但不深入架构细节 |
| 二、四个核心命令 | 完整覆盖（核心） | MVP 的全部教学主体 |
| 三、自动化机制 | 仅口头提及 | 属于进阶内容，一句话带过即可 |
| 四、数据文件说明 | 不覆盖 | 用户无需手动操作数据文件 |
| 五、典型工作日 | 融入结尾总结 | 用 15 秒快速串联一天的工作流 |
| 六、智能化特性 | 不覆盖 | 属于进阶内容，留给后续视频 |
| 七、常见问题 | 不覆盖 | 引导用户查阅 HTML 手册 |

### 知识点排序（学习路径）

```
捕获（输入）→ 规划（处理）→ 时间块（执行）→ 回顾（反馈）
   /capture     /plan-day     /time-block      /weekly-review
```

这个顺序遵循 PDCA 循环逻辑：先有输入，再做规划，然后落地执行，最后回顾改进。每个命令自然衔接下一个，形成闭环。

### 评估标准

- 视频发布后观察：用户是否在 1 周内开始使用 `/capture` 和 `/plan-day`
- 收集反馈：是否需要更详细的进阶视频（自动化机制、智能化特性）

## 制作备注

### 终端美化要求

- **终端主题**：深色背景（#0f172a 或类似），与 SPE 使用手册配色一致
- **字体**：Cascadia Code 或 JetBrains Mono，16-18px
- **光标**：块状光标，青色（#14b8a6）
- **命令输入速度**：模拟真人打字，约 60-80 字符/秒，关键命令处稍慢
- **输出显示**：逐行快速出现（非瞬间全部显示），模拟终端输出感

### 旁白要求

- **语速**：中等偏快（约 250 字/分钟中文），保持节奏感
- **语气**：专业但不刻板，像一个资深同事在给你演示工具
- **停顿标记**：脚本中 `...` 表示短停顿（0.3-0.5 秒），用于节奏控制
- **TTS 工具建议**：Azure TTS（zh-CN-YunxiNeural）或 ElevenLabs 中文模型

### 侧边信息图清单

需要预制以下静态图/动画素材：

| 编号 | 素材 | 出现场景 | 说明 |
|------|------|----------|------|
| G1 | SPE 四层架构图 | 场景 2、场景 6 | 捕获→处理→执行→回顾，四层竖向排列 |
| G2 | 收件箱分流示意图 | 场景 3 | 一个收件箱 → 四条箭头 → 决策/委派/个人/参考 |
| G3 | 精力分配时段图 | 场景 4 | 横向时间轴，上午高精力到傍晚低精力的渐变 |
| G4 | PDCA 循环图 | 场景 5 | 四象限循环，回顾→规划箭头高亮 |
| G5 | 四命令进度指示器 | 全程 | 左上角常驻，当前命令高亮 |
| G6 | 颜色编码速查 | 场景 4 | 四个色块：青色(Focus)/黄色(常规)/红色(Deadline)/绿色(学习) + 说明文字 |

### 录制注意事项

1. **终端窗口尺寸**：1920x1080，终端占据画面 70% 左宽，右侧 30% 留给侧边信息图
2. **所有命令输出均为预制文本**：不依赖实际 API 响应，确保录制可控
3. **每个命令演示后停顿 1 秒**：让观众消化输出内容
4. **侧边信息图使用半透明背景**：不遮挡终端内容
5. **转场统一使用滑动过渡**：保持视觉连贯性，避免花哨特效
