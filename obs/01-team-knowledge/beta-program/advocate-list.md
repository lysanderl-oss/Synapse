## 使用说明

1. **提名来源**：Lysander 从个人网络 / 已合作案例 / 社区观察中挑选
2. **Founding Number 分配**：按"接受 Beta"时间顺序自动分配 #001, #002, ...
3. **状态流转**：
   - `未联系` → `已邀请` → `已接受` / `已拒绝` → `已部署` → `反馈中` → `案例化`
   - 任一阶段可回落到 `流失`（30 天无活动）
4. **字段必填**：团队名 / 规模 / 联系方式 / 承包人 / 状态
5. **隐私**：本文件留在 Synapse-Mini 私有仓库，不进入公开 OBS 分享

## 状态字段定义

| 状态 | 触发条件 | 下一步 |
|------|----------|--------|
| `未联系` | 名字进入观察池但尚未发出邀请 | Lysander 决定邀请时机 |
| `已邀请` | 已发出个性化邀请邮件/DM | 等待 3–7 天回应 |
| `已接受` | 对方同意加入 Founding 30 | 分配 Founding # + 发欢迎邮件 |
| `已拒绝` | 对方明确拒绝或 14 天无回应 | 标记后移除出活跃名单 |
| `已部署` | 72 小时部署 Check 通过 | 进入 Week 1 onboarding |
| `反馈中` | 已提交至少 1 次结构化反馈 | 正常运营 |
| `案例化` | 已授权公开引用或署名案例 | 纳入 `/synapse/case-studies` |
| `流失` | 30 天无活动 | Lysander 发轻量关怀邮件，否则移出 |
