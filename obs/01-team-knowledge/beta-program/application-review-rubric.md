## 使用流程

```
申请邮件到达 (Gmail label: synapse-beta-apply)
  ↓
growth_lead 打开此表拷贝 "空白条目" 模板
  ↓
5 维度逐项评分（1–5 分）
  ↓
总分自动 = 5 维之和（最高 25）
  ↓
决策：
  ≥ 16 → 进入 Beta，回复欢迎邮件
  13–15 → 列入"观察池"，2 个月后复审
  ≤ 12 → 礼貌拒绝，保留联系方式
  ↓
结果记录到 advocate-list.md + 本文件末尾评分日志
```

## 5 维度评分表

### 维度 1：契合度（Fit）— 1-5 分

**衡量：申请方与 Synapse 核心价值主张的契合程度**

| 分数 | 判断标准 |
|------|----------|
| 5 | 完全契合：多人团队 + 已用 Claude Code + 有明确 AI ops 治理痛点 + 行业匹配（AI Builder / Tech Team / 建筑数字化 / 量化） |
| 4 | 高度契合：符合 3/4 上述特征，缺 1 项可补 |
| 3 | 基本契合：符合 2/4，需要进一步沟通判断真实场景 |
| 2 | 弱契合：仅符合 1/4，属于"听说过 Claude Code 想试试"层面 |
| 1 | 不契合：与 AI 工作流基本无关，或只想要免费订阅 |

### 维度 2：使用深度预期（Depth Potential）— 1-5 分

**衡量：申请方未来投入 Synapse 使用的深度（部署、改造、长期运营）**

| 分数 | 判断标准 |
|------|----------|
| 5 | 明确表示"要在团队全面推广"，或已描述具体落地场景 |
| 4 | 有明确某一场景要用 Synapse，计划具体 |
| 3 | 说了想用但场景模糊，需要 onboarding 时帮其澄清 |
| 2 | 只想"探索"、"了解一下"，无实际使用计划 |
| 1 | 完全无使用计划，只是好奇 |

### 维度 3：反馈能力（Feedback Capacity）— 1-5 分

**衡量：申请方能否提供高质量、结构化反馈**

| 分数 | 判断标准 |
|------|----------|
| 5 | 申请信就包含对 Synapse 的具体观察 / 质疑 / 改进建议（证明有判断力） |
| 4 | 申请人有技术背景 + 沟通清晰，文字表达专业 |
| 3 | 能沟通但较简短，可能需要引导才能给出深度反馈 |
| 2 | 沟通能力一般，反馈预期会比较表面 |
| 1 | 申请信一句话就结束，或答非所问 |

### 维度 4：案例潜力（Case Potential）— 1-5 分

**衡量：该团队若使用成功，能否产出可公开引用的案例**

| 分数 | 判断标准 |
|------|----------|
| 5 | 知名团队或有趣细分场景，案例化后对 Synapse 品牌有显著加分 |
| 4 | 中等知名度或特色场景，可匿名引用 |
| 3 | 一般团队，案例价值有限但总比没有好 |
| 2 | 极小众场景，难以泛化 |
| 1 | 无案例价值 |

### 维度 5：推荐价值（Referral Value）— 1-5 分

**衡量：该团队能否带来其他优质 founding 候选的推荐**

| 分数 | 判断标准 |
|------|----------|
| 5 | 有明确社群影响力（KOL、社区 maintainer、畅销作者） |
| 4 | 活跃于某细分圈子，有一定网络 |
| 3 | 普通专业人士，有限推荐能力 |
| 2 | 网络弱，主要是个人使用 |
| 1 | 基本无推荐潜力 |

## 决策边界

| 总分 | 决策 | 操作 |
|------|------|------|
| **23–25** | **A+ 优先录取** | 立即发欢迎邮件，Lysander 亲自签名；争取作为首批种子案例 |
| **19–22** | **A 录取** | 发欢迎邮件，标准 onboarding |
| **16–18** | **B 录取** | 发欢迎邮件，但加一句 "期待你提供深度反馈" 的提醒 |
| **13–15** | **观察池** | 回邮 "Founding 30 席位有限，我们把你放入下一批候选" + 2 个月复审 |
| **≤ 12** | **礼貌拒绝** | 回邮 "谢谢申请，Synapse 开源免费可直接使用，欢迎后续以普通用户身份加入社区" |

## 特殊规则

1. **Lysander 直接指定**：无论分数，Lysander 可以手动指定进入 Beta（例如老朋友、战略合作），记为 "指定录取"
2. **第 31 位之后**：一旦 Founding 30 满员，新申请自动进入"观察池"，等待阶段 C
3. **分数 = 16 边界线**：落在 16 分的申请 Lysander 二次复核决定
4. **Rubric 本身的迭代**：每月评审一次 Rubric 是否需要调整（由 growth_lead 发起）
