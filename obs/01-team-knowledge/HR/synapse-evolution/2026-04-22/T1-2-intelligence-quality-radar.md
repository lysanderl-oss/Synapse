# 情报质量雷达图 Intelligence Quality Radar

## 一、概述

情报质量雷达图是 Synapse 情报管线的统一评估标准，用于对情报日报、情报行动等输出进行多维度质量评分。

**使用场景：**
- 情报日报发布前的自检与评分
- 情报行动建议的优先级排序
- 情报管线迭代优化的数据基准
- Agent 输出质量的量化考核

**评估频率：** 每次情报输出必评，月末汇总管线健康度。

---

## 二、评分维度定义

### 维度1：准确性 Accuracy（权重：25%）

情报内容是否正确，有无事实错误、逻辑漏洞或夸大表述。

**评分标准（0-5分）：**

| 分数 | 标准 |
|------|------|
| 5分 | 信息来源权威（一手信源/官方发布/学术论文），数据精确，零事实错误，逻辑自洽 |
| 4分 | 来源可靠，主体信息正确，有少量边缘信息未经交叉验证但无已知错误 |
| 3分 | 核心信息正确，存在1-2处未经充分验证的引用或数据，错误不影响结论 |
| 2分 | 有明确事实错误或数据矛盾1-2处，需读者自行辨别 |
| 1分 | 存在重大事实错误或多处来源矛盾，无法独立使用 |

**评估方法：**
- 逐篇核查引用来源可信度
- 交叉验证关键数据点（至少3个来源）
- 检查是否存在夸大、误导性表述

**数据来源：** 来源URL列表、信源评级数据库、内部信源黑名单

---

### 维度2：实用性 Actionability（权重：30%）

情报是否包含明确、可执行的建议，以及执行路径是否清晰。

**评分标准（0-5分）：**

| 分数 | 标准 |
|------|------|
| 5分 | 每条情报附具体行动建议（Who/What/When明确），可直接转化为任务 |
| 4分 | 有明确建议方向，执行路径部分明确（70%以上要素齐全） |
| 3分 | 建议存在但泛化（如"关注AI进展"），需二次解读才能执行 |
| 2分 | 仅描述现象，无明确建议，或建议与情报结论脱节 |
| 1分 | 纯信息罗列，无任何行动指向 |

**评估方法：**
- 检查情报是否包含"建议"或"行动"章节
- 验证建议与情报内容的逻辑关联度
- 评估执行路径完整性（执行者、目标、触发条件）

**数据来源：** 情报正文、行动建议模块、执行日志反馈

---

### 维度3：时效性 Recency（权重：15%）

情报是否足够新，是否在有效信息窗口期内。

**评分标准（0-5分）：**

| 分数 | 标准 |
|------|------|
| 5分 | 事件发生 ≤ 24小时内，信息窗口黄金期 |
| 4分 | 事件发生 1-3天内，仍处价值高峰期 |
| 3分 | 事件发生 3-7天内，基础价值仍在 |
| 2分 | 事件发生 7-14天，属于历史回顾而非即时情报 |
| 1分 | 事件发生 > 14天，或使用过时数据（无注明） |

**评估方法：**
- 记录情报覆盖事件的最新时间节点
- 对比情报发布与事件发生的间隔
- 标注信息衰减阶段（新鲜期/稳定期/过期候选）

**数据来源：** 情报时间戳、事件时间节点、信息衰减跟踪表

---

### 维度4：相关性 Relevance（权重：15%）

情报与 Synapse / Janus Digital 核心业务的关联程度。

**评分标准（0-5分）：**

| 分数 | 标准 |
|------|------|
| 5分 | 直接影响业务决策，或与当前重点项目高度相关，可立即应用 |
| 4分 | 属于相关领域动态，关联度较高（>70%），有潜在应用场景 |
| 3分 | 行业相关但间接（30-70%），需转化才能落地 |
| 2分 | 泛行业信息，关联度低（10-30%），需深度解读 |
| 1分 | 与Synapse/Janus Digital业务几乎无关联（<10%） |

**评估方法：**
- 标签匹配：情报标签与业务关键词库匹配率
- 人工判断：按业务影响度评估（由 Lysander 或智囊团裁定）
- 回顾验证：跟踪情报是否在后续业务讨论中被引用

**数据来源：** 业务关键词库、业务引用跟踪日志

---

### 维度5：完整性 Completeness（权重：15%）

情报是否包含背景、影响、建议等完整要素，结构是否规范。

**评分标准（0-5分）：**

| 分数 | 标准 |
|------|------|
| 5分 | 包含：背景(What)、影响(Why)、来源(Source)、建议(Action)全四要素，结构清晰 |
| 4分 | 包含3个核心要素，结构基本完整 |
| 3分 | 包含2个核心要素，内容覆盖尚可 |
| 2分 | 仅包含1个核心要素，内容单薄 |
| 1分 | 零要素，碎片化信息，无法独立成篇 |

**评估方法：**
- 模板检查：对照情报标准模板（背景-影响-来源-建议）逐项核查
- 字数评估：正文 ≥ 300字为完整基准
- 要素覆盖率 = 已含要素数 / 4要素 × 100%

**数据来源：** 情报结构评分表、字数统计工具

## 三、雷达图计算公式

**维度得分计算：**
```
维度原始分 = Σ(各评分项得分) / 评分项数量
维度标准化分 = 维度原始分 / 5 × 100
```

**加权总分（0-100分）：**
```
总分 = (准确性得分 × 25%) + (实用性得分 × 30%) + (时效性得分 × 15%) + (相关性得分 × 15%) + (完整性得分 × 15%)
```

**评级标准：**
| 总分区间 | 评级 | 说明 |
|----------|------|------|
| 85-100 | A 卓越 | 可直接发布，值得归档 |
| 70-84 | B 优秀 | 质量良好，少量优化即可发布 |
| 55-69 | C 合格 | 基本可用，需补充关键要素 |
| 40-54 | D 不足 | 需大幅修改，实用性或准确性存在明显短板 |
| 0-39 | F 不合格 | 不建议发布，需重新制作 |

**雷达图坐标：**
五个维度等角度分布（每轴72度），得分连接成不规则多边形，闭合面积越大质量越高。

## 四、历史基准

基于2026年4月情报日报样本的初步估算：

| 维度 | 当前平均得分（估算） | 说明 |
|------|---------------------|------|
| 准确性 Accuracy | 3.2 / 5（64分） | 主要依赖二手信源，缺乏交叉验证 |
| 实用性 Actionability | 2.8 / 5（56分） | 建议方向有但执行路径不够具体 |
| 时效性 Recency | 4.0 / 5（80分） | 管线执行及时，事件覆盖较新 |
| 相关性 Relevance | 3.0 / 5（60分） | 话题广泛但与业务结合度待提升 |
| 完整性 Completeness | 3.5 / 5（70分） | 结构较完整，背景和影响描述较好 |

**当前管线加权总分估算：62分（C级合格）**

改进优先级：实用性 > 准确性 > 相关性 > 完整性 > 时效性

## 五、持续改进机制

### 评分反馈闭环
```
情报发布 → 评分员评分 → 记录至情报评分数据库
         → 发现短板 → 反馈至情报采集Agent
         → 月度汇总 → 生成管线健康度报告
```

### 评分员制度
- 日常评分：情报行动Agent自动评分 + knowledge_engineer复核
- 月度审计：harness_engineer执行管线审查，输出评分分布报告
- 季度校准：根据实际业务引用情况校准相关性权重

### 评分驱动优化
- **实用性 < 3分**：强制要求情报行动Agent在建议模块增加执行路径
- **准确性 < 3分**：将该来源标记为低可信度，优先使用一手信源
- **相关性 < 3分**：调整关键词采集过滤规则，提高业务相关性
- **完整性 < 3分**：强制执行情报模板检查，未通过不得发布

### 评分透明化
- 每篇情报发布时附评分摘要（雷达图缩略图 + 总分）
- 月度情报汇总报告包含：平均分、雷达图趋势、各维度分位数
- 低于C级（<55分）的情报自动归档为"待改进"状态，不计入管线健康度
