## 二、Graphify团队职责

### 2.1 战略分析师 (strategist)

**核心职责**：
- 复杂问题诊断和拆解
- SWOT、PEST等战略框架分析
- 方案设计和路径规划
- 风险识别和应对

**输出**：
- 问题分析报告
- 战略选项对比
- 风险评估矩阵

### 2.2 关联发现专家 (relation_discovery)

**核心职责**：
- 知识图谱构建
- 隐性关联发现（Graphify核心）
- 跨领域知识链接
- 图推理查询

**输出**：
- 关联路径分析
- 知识链接建议
- 图谱可视化

### 2.3 趋势洞察师 (trend_watcher)

**核心职责**：
- 时间序列趋势分析
- 模式识别和发现
- 异常检测和预警
- 预测建模

**输出**：
- 趋势分析报告
- 模式发现洞察
- 预测建议

### 2.4 决策顾问 (decision_advisor)

**核心职责**：
- 多选项评估
- 利弊量化分析
- 敏感性分析
- 行动建议生成

**输出**：
- 决策评估报告
- 利弊平衡表
- 明确行动建议

## 四、调用场景

## 五、Graphify使用指南

### 5.1 召唤方式

```
# 直接召唤Graphify智囊团
lysander 帮我思考一下这个项目的问题

# 召唤特定专家
lysander 用关联发现专家帮我找找这个问题和什么的关联

# 召唤多个专家协作
lysander 需要战略分析和趋势洞察
```

### 5.2 最佳实践

```
✓ 清晰描述背景和目标
✓ 说明你希望得到什么类型的洞察
✓ 允许Graphify深入分析（不催促）
✓ 对洞察进行反馈，帮助Graphify学习

✗ 不要问模糊的问题（"帮我分析一下"）
✗ 不要期望立即得到答案（深度分析需要时间）
✗ 不要忽略关联发现（最有价值的部分）
```

### 5.3 反馈机制

```
你收到Graphify的洞察后：
1. ✅ 有价值 → "这个分析很有用"
2. ❌ 不相关 → "这个关联不太对，需要关注X方面"
3. 🤔 需要深入 → "这个点可以再深挖吗"

反馈会被OBS知识团队记录，用于优化后续分析
```

## 七、与其他团队的协作

### 7.1 Graphify → 执行层

```
Graphify发现洞察 → 派发给执行团队

示例：
Graphify: "发现IoT项目交付周期长的根本原因是..."
  → 派发给 Butler团队 (delivery_expert) 优化流程
  → 派发给 RD团队 (tech_lead) 优化技术方案
```

### 7.2 执行层 → Graphify

```
执行团队遇到复杂问题 → Graphify提供深度分析

示例：
Butler团队: "有个复杂交付问题难以解决"
  → Graphify分析 → 发现关联风险 + 给出建议
```

### 7.3 Graphify ↔ OBS

```
Graphify是OBS的"用户"，OBS是Graphify的"知识源"

Graphify → OBS:
  - 读取沉淀的知识
  - 构建知识图谱
  - 关联历史案例

OBS → Graphify:
  - 接收新洞察归档
  - 接收反馈优化
  - 接收图谱更新
```
