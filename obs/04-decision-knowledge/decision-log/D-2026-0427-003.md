# D-2026-0427-003 — CLAUDE.md edit checklist enforced via git pre-commit hook

## 背景

2026-04-27 morning, Lysander 沉淀了 5 维度 CLAUDE.md 修改前事实校验 checklist
(`.claude/harness/claude-md-edit-checklist.md`)。但 checklist 仅作为 Markdown 文档存在，
依赖 Lysander 自觉执行。当日下午总裁挑战："机制不能只是说说而已" — checklist 没有
强制约束力，Lysander 跳过它的成本为零。

## 决策

将 5 维度 checklist 转化为 git pre-commit hook，commit 时自动校验，违规即拦截：

1. **新增 `scripts/check_claude_md_edit.py`** — 5 维度自动检查
   - A: 数字 vs `agent-CEO/config/organization.yaml` SSOT (44/46/50 forbidden)
   - B: 路径存在（`agent-butler/*` 已废弃 / `.claude/harness/` 碎片必须存在）
   - C: 函数与 Skill 引用必须有实现（`audit_harness` / `/dispatch`）
   - D: 行数 ≤ 350
   - E: QA 阈值 = 85/100（不再用 3.5）

2. **新增 `scripts/install-git-hooks.sh`** — 幂等安装器
   - 写入 `.git/hooks/pre-commit`
   - 串联 `check_claude_md_edit` + `check_decision_log` + `frontmatter_lint`

3. **更新 `.claude/harness/claude-md-edit-checklist.md`** — 追加"实际执行机制"章节
   - 安装步骤 / 拦截行为 / 测试用例 / 局限 / 紧急绕过政策

## 验证

| 故意破坏 | 期望 | 实测 |
|---------|------|------|
| 注入 `agent-butler/` 路径 | exit 1 | ✅ exit 1，命中 B-FAIL |
| 注入 `44 人 Agent` | exit 1 + 3 个 A-FAIL | ✅ exit 1，A 全部命中 |
| 干净状态 | exit 0 | ✅ exit 0 |
| `git commit` 触发 hook | 拦截带破坏的 commit | ✅ 实测拦截 |

## 局限

- pre-commit hook 仅本地，不影响远端推送（远端无对应 CI gate）
- `git commit --no-verify` 可绕过 — 属"自律层"，需 24h 内由 Lysander 在
  `obs/06-daily-reports/` 复盘
- frontmatter_lint 仍 warning 模式，不阻断（计划下一轮升级到 block）
- 已发现 `check_decision_log.py` 在 Windows git `-m` 模式下读取的是 HEAD 而非
  proposed message — 后续单独修复（不在本决策范围）

## 问责

Lysander 4-27 morning 沉淀的 checklist 是纸面机制，evening 在总裁警觉下
转化为代码执行。后续所有"沉淀型"机制必须在沉淀同日转化为 hook / lint /
CI 等强制执行手段，否则记入周审"纸面机制"清单。
