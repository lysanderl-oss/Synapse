# 网站视觉国际化方案 v2

| 页面 | 当前配图 | 状态 |
|------|:--------:|:----:|

| /services | 0 | ❌ 空白 |
| /training | 0 | ❌ 空白 |
| /intelligence | 0 | ❌ 空白 |
| /about | 0 | ❌ 空白 |
| /blog | 0 | ❌ 空白 |

### 问题2：漫画风格与国际化定位不兼容

| 维度 | 当前漫画 | 国际标杆(Linear/Vercel/Stripe) |
|------|----------|-------------------------------|
| 风格 | Q版卡通角色 | 抽象几何 / 产品截图 |
| 感觉 | 可爱但不专业 | 克制、精致、专业 |
| 信息密度 | 一张图塞太多内容 | 单一焦点、极度简洁 |
| 文化感 | 亚洲卡通风格 | 去文化特征、全球通用 |
| 标杆案例 | 无一使用漫画 | 全部使用抽象或产品截图 |

## 二、推荐方案：抽象几何 + 产品截图混合

### 战略原则

```
从标杆提炼的4条原则：
1. 少即是多 — 不需要更多图，需要更精准的图
2. 产品即视觉 — 真实截图比任何插画都有说服力
3. 功能即美学 — 每个视觉元素都传递信息，不是装饰
4. 去文化特征 — 几何抽象 = 全球通用
```

### 具体方案

#### Hero 区（首页顶部）

**删除漫画**，改为：
- 纯品牌色渐变背景（Dark → Deep Blue 渐变）
- 右侧：一个精致的抽象几何图形（品牌色的连接节点/网络图，呼应Synapse突触概念）
- 用CSS/SVG实现，不用外部图片
- 参考：Linear 的 Hero（纯渐变+微妙动画）

#### 各 Section 背景

**删除所有漫画底图**，改为：
- 白色/浅灰交替 + 品牌色渐变分割线
- 偶尔一个section使用极淡的品牌色渐变背景
- 不需要图片 — 用排版和留白创造节奏

#### 信任信号区

**保留数据卡片**，新增：
- 一张 operation.html 的浏览器窗口截图（展示评审过程的真实界面）
- 截图放在 macOS 风格的浏览器框内（三个圆点+地址栏），增加专业感

#### 服务页

每个服务卡片新增一个小型"产品预览"：
- Assessment：展示审计评分界面截图
- Implementation：展示 CLAUDE.md 配置文件截图
- Training：展示课程大纲结构截图

#### 培训页

- 课程模块列表旁边：一张 Lab 实操界面截图

#### About 页

- 架构图保留（代码渲染的架构文本，已经很专业）
- 不加图片

#### 情报页

- 展示一张情报日报HTML的浏览器窗口截图

### 视觉资产生成方式

| 类型 | 工具 | 说明 |
|------|------|------|
| 抽象几何背景 | CSS/SVG代码 | 品牌色渐变+节点网络图 |
| 产品截图 | 浏览器截图 | 截取真实运行的HTML页面 |
| 浏览器窗口框 | CSS | macOS风格圆点+地址栏 |
| 渐变分割线 | CSS | 已有 glow-line 类 |

**不再使用AI生成漫画。**

### 国际化处理

| 维度 | 当前 | 优化后 |
|------|------|--------|
| 视觉风格 | 漫画卡通 | 抽象几何 + 产品截图 |
| 网站语言 | 纯中文 | 中文为主（后续考虑双语） |
| 设计语言 | 混杂 | 统一极简商务 |
| 字体 | Inter（已国际化） | 保持 |
| 色彩 | Janus品牌色（已国际化） | 保持 |
