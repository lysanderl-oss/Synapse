## 本版本 Bug 修复记录

### BUG-001 (P1) — WF-01 "邀请团队成员" 节点失败 ✓ 已修复（v2.4.0）

- **现象**：WF-01 执行在"邀请团队成员加入项目"节点失败，`TypeError: this.getCredentials is not a function`
- **根因**：n8n Code Node 在 Task Runner 模式下无法调用 `this.getCredentials()`（n8n 平台限制）
- **修复**：将 Code Node 替换为 HTTP Request 节点链（get-team-members-v2 → filter-member-gids-v2 → add-project-members-v2）
- **验证**：exec #13731 (2026-04-27T12:20:46Z) status:success，Asana GID=1214313881117253 创建，所有节点通过

### BUG-002 (P1) — WF-08 Webhook 节点立即 error ✓ 已修复（v2.4.0）

- **现象**：WF-08 exec 13706 Webhook 节点立即 error，`TypeError: Cannot read properties of undefined (reading 'name')`
- **根因**：connections dict 存在 stale key "汇总发送监控频道"，导致 n8n 内部 Webhook 响应模式配置检查访问 undefined
- **修复**：stale key 重命名为 "Notify via WF-09 (was 汇总发送监控频道)"
- **验证**：exec #13724、#13722 均 status:success，响应时间 ~1.4s

### BUG-005 (P1) — WF-01 仅 PM 加入 Asana，DE/SA/CDE 缺失 ✓ 已修复（v2.5.0）

- **现象**：WF-01 执行后 Asana 项目仅 PM 1人加入，DE/SA/CDE 三名成员缺失
- **根因**：WF-01 "提取注册表数据" Code Node 仅提取 pmEmail，未提取 deEmail/saEmail/cdeEmail
- **修复**：新增三字段提取逻辑，从 Notion props 中读取并传递至下游邀请节点
- **验证**：0427C TC-A01 BUG-005 PASS，Asana 项目 GID=1214320092268919 成员 4 人（PM+DE+SA+CDE 全部加入）

### BUG-006 (P1) — WF-01 执行后 `团队信息维护` 字段被重置为"未维护"（回归）✓ 已修复（v2.5.0）

- **现象**：WF-01 执行后，Notion 页面 `团队信息维护` 字段从"已维护"被重置为"未维护"
- **根因**：WF-01 "回填注册表链接"节点硬编码写入 `status: { name: '未维护' }`（v2.x 开发中回归引入，v1.5.4 已修复过）
- **修复**：删除该字段写入操作
- **验证**：0427C TC-A01 BUG-006 PASS，BEFORE/AFTER 均为"已维护"

### BUG-003 (P1) — WF-12 (WBS工序导入) 从未执行，WBS 导入链路断裂 ✓ 已修复（v2.6.0，本版本）

- **现象**：WF-12 (id: p8tPxmkhMcQPcRMh) 零执行历史，WF-11→pmo-api→WF-12 链路断裂，Asana 项目 0 任务
- **根因**：WF-12 处于 inactive 状态，webhook path `pmo-wf02-wbs-import` 注册 404，pmo-api 回调不可达
- **修复**：通过 n8n API 将 WF-12 激活为 active=true，webhook 重新注册生效
- **验证**：TC-A02 PASS — 0427C 项目（Asana GID 1214320092268919）导入 80 节点（13主任务+67子任务），WF-11 exec 13897，pmo-api job 731503d8，wbs_success=true，hub_success=true

### BUG-004 (P2) — DE/SA/CDE 邮箱字段未提取 → INVALID

- **原始评估**：Code Node 未提取 deEmail/saEmail/cdeEmail（后确认为 BUG-005，已在 v2.5.0 修复）
- **INVALID 原因**：测试批次 0427 使用了 `团队信息维护=未维护` 的不合规测试数据（违反 PRINCIPLE-002），邮箱为空是预期行为，并非 Bug
- **结论**：此编号 INVALID，BUG-005 为对应的真实缺陷
